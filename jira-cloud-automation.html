Project

Jira Cloud Automation

Automation toolkit for Jira Cloud focused on configuration discovery, migration support, and governance for large-scale instances.

Overview

This project groups the automation work I do around Jira Cloud using ScriptRunner (Groovy), Python, and the Jira REST API. The goal is to replace manual, one-off admin tasks with repeatable, documented, and safe jobs that help during migrations, cleanups, and day-to-day operations.

The examples are anonymized but based on real scenarios: custom fields inventory, cleanup orchestration, demo issue generation, and guardrails for configuration changes.

Jira Cloud · ScriptRunner · Automation REST APIs · Python · JSON

Key responsibilities

Design automation patterns for Jira Cloud

ScriptRunner · Groovy · Governance

  • Create jobs that scan configuration (custom fields, screens, workflows, projects) and export structured inventories to CSV/JSON.
  • Define conventions, guardrails, and “safe defaults” so admins can run automation against production instances with confidence.
  • Document each job with input parameters, scope, and expected output, making it easy to reuse across clients or environments.

Build Python helpers around the Jira REST API

Python · REST · Tooling

  • Develop small command-line tools to perform bulk reads and updates when ScriptRunner alone is not enough.
  • Use API pagination, authentication, and error handling patterns that are robust and easy to extend.
  • Keep scripts versioned in GitHub and aligned with ScriptRunner code and documentation.

Support migrations and cleanups

Data Center → Cloud · Configuration hygiene

  • Provide inventories and reports that help decide which configuration elements can be removed, merged, or migrated.
  • Design “read-only” patterns for projects during cut-over windows to avoid accidental changes.
  • Coordinate dry-runs, logging, and rollback options so changes are auditable and reversible.

Highlighted automations

Jira Cloud Custom Fields Inventory (view page)

ScriptRunner · Reporting · Cleanup support

  • ScriptRunner job that exports a detailed inventory of custom fields, including type, contexts, usage notes, and comments for fields that could not be fully resolved.
  • Output feeds Excel/Python analysis to identify duplicates, unused fields, and consolidation opportunities before a migration.

Weekly Issue Generator (view page)

Scheduling · Demo environments · Monitoring

  • Scheduled job that automatically creates a sample issue in a demo project, used to showcase workflows and automation rules.
  • Demonstrates logging, idempotency, and parameterization (project, issue type, labels) while keeping the script simple to understand.

Field cleanup orchestrator (concept)

Orchestration · Incremental rollout

  • Proposed architecture that splits a large “cleanup script” into smaller jobs: inventory, analysis, and change roll-out.
  • Each phase produces outputs that the next phase reads, making the process easier to test, pause, or roll back.
