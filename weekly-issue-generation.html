WhatsApp API Automation

WhatsApp Conversational Ordering Bot (“Chavela”)
November 2025

I built a WhatsApp-based ordering assistant to standardize customer order intake, reduce repetitive follow-ups, and minimize manual coordination. The system converts unstructured chat messages into structured, confirmed orders, then alerts the administrator only when action is required.

Problem

  • Orders arrived across multiple channels (WhatsApp, social DMs, ad-hoc messages) with no single source of truth.
  • A person had to monitor and respond continuously, limiting scalability.
  • Follow-ups and clarifications consumed significant time and increased the risk of errors.
  • Menus and prices became inconsistent across platforms due to manual updates.

Solution

  • Implemented a guided conversational flow for ordering, including menu access, order capture, summary, and confirmation.
  • Built a rule-based parsing layer to interpret free-text orders with spelling variations and informal language.
  • Added confirmation-based submission to reduce errors and prevent incomplete orders.
  • Triggered real-time admin notifications only after user confirmation.

Architecture

  • Messaging: Twilio WhatsApp Business API for inbound/outbound messages.
  • Webhook API: receives messages and responds with the next conversational step.
  • State Machine: deterministic flow control (menu → capture → summary → confirmation).
  • Parsing & Pricing: regex + dictionaries + fuzzy matching for robust extraction.
  • Persistence: stores conversation state and confirmed orders (SQLite / PostgreSQL-ready).
  • Notifications: sends a structured order summary to the administrator upon confirmation.

Key Features

  • Keyword-triggered menu access (e.g., “menu”, numeric options).
  • Free-text order parsing for quantities, products, and variants.
  • Automatic order summarization and total calculation.
  • Explicit confirmation step before order submission.
  • Admin alert with clean, structured order details.

Edge Case Handling

  • Misspellings and informal language (e.g., product typos).
  • Multiple items in a single message.
  • Ambiguous quantities resolved via clarification prompts.
  • Duplicate confirmations handled with idempotency checks.
  • Session timeout and context reset after inactivity.

Outcome

  • Reduced manual effort by standardizing order intake and confirmations.
  • Improved accuracy through confirmation-based submission and structured summaries.
  • Reduced back-and-forth follow-ups by consolidating information into a single confirmation step.
  • Created a scalable foundation for future improvements (catalog management, admin dashboard, analytics).

Future Improvements

  • Centralized product & pricing catalog to keep menus consistent across platforms.
  • Admin dashboard for order monitoring and operational analytics.
  • Parsing confidence scoring with human fallback for ambiguous cases.
  • Scheduled/broadcast messaging using approved WhatsApp templates.
  • Support additional intake channels beyond WhatsApp.
