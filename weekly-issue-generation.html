WhatsApp Ordering Assistant – Chavela

A deterministic WhatsApp-based ordering assistant that converts unstructured chat messages into structured, confirmed orders—reducing manual follow-up and operational friction.

Twilio WhatsApp API State Machine Order Parsing & Pricing
Problem

Operational bottlenecks in multi-channel ordering

Order intake was handled through multiple informal channels, creating operational friction and unnecessary manual work.

  • Orders coming from multiple channels: customer requests arrived via WhatsApp, social DMs, and ad-hoc messages, with no centralized system.
  • Constant manual availability: someone had to monitor and reply continuously to interpret messages and confirm orders.
  • Time lost in follow-ups: confirmations, clarifications, and status tracking required repeated back-and-forth communication.
  • Menus out of sync: product lists and prices were updated manually across platforms, causing inconsistencies and outdated information.
Idea

Standardize ordering via a guided conversational flow

I built a WhatsApp-based ordering assistant that standardizes the full order intake flow while keeping the interaction natural for customers. The assistant guides users through a clear conversation, parses free-text orders, generates a structured summary with total cost, and requires explicit confirmation before final submission.

  • Guided steps to reduce ambiguity and customer friction
  • Free-text parsing with support for informal language and spelling variations
  • Confirmation-based submission to reduce errors
  • Admin notification only when action is needed
Architecture

Lightweight, modular, event-driven backend

The system is built as a webhook-driven service with a deterministic state machine to ensure predictable behavior, easier debugging, and low operational cost.

  • Messaging: Twilio WhatsApp Business API (inbound/outbound)
  • Webhook API: receives messages and returns replies
  • State Machine: controls flow and user context (menu → capture → summary → confirmation)
  • Parsing Layer: rule-based NLP (regex, dictionaries, fuzzy matching)
  • Persistence: stores conversation state and confirmed orders
  • Admin Notifications: sends structured orders upon confirmation
Core Features

Reliable ordering, built for real-world messages

  • Keyword-triggered menu access (e.g., “menu”, numeric options)
  • Order parsing for quantities, products, and variants from unstructured text
  • Automatic pricing and order summarization
  • Explicit confirmation step before final submission
  • Admin alert with clean, structured order details
Edge Cases

Designed for messy inputs and operational safety

  • Misspellings and informal language (“chilaqul”, “snakies”)
  • Multiple items expressed in one message
  • Ambiguous quantities with clarification prompts
  • Duplicate confirmations (idempotency)
  • Session timeout and context reset after inactivity
Results

Lower manual work, fewer errors, more scalability

  • Reduced manual effort by standardizing order intake
  • Improved accuracy through confirmation-based submission
  • Lowered follow-up loops via structured summaries
  • Created a scalable foundation for future channels and an admin dashboard
Future Improvements

Extendability and operational tooling

  • Centralized product & pricing catalog
  • Admin dashboard for order monitoring and analytics
  • Parsing confidence scoring with human fallback
  • Scheduled/broadcast messaging using approved templates
  • Support additional intake channels beyond WhatsApp
