Jira Weekly Issue Generator

Problem

Teams needed recurring weekly Jira issues (status updates, operational checklists, governance tasks) created consistently across multiple projects. When handled manually, tickets were often created late, assigned inconsistently, duplicated, or missing key fields—leading to confusion, broken reporting, and extra coordination work for admins and project leads.

Common pain points
  • Weekly tickets created late (or forgotten) during peak workload weeks.
  • Inconsistent fields (components, labels, due dates, assignees) breaking dashboards.
  • Duplicate issues across projects or incorrect issue types.
  • No audit trail of what was created, skipped, or failed.
Main idea

Implement a scheduled ScriptRunner (Groovy) automation that generates weekly issues using a controlled template. The job runs on a cadence, queries Jira using JQL to identify the scope (projects/boards/owners), creates the issues with standardized metadata, and writes logs/outputs for traceability—ensuring consistency without manual overhead.

What the automation handles
  • Scheduling: runs weekly via ScriptRunner job scheduler (no manual triggers).
  • JQL targeting: identifies which projects/boards need weekly issues.
  • Standardization: sets issue type, summary format, labels, components, and due dates.
  • Assignments: routes tickets to owners (or fallback assignee rules).
  • Idempotency: avoids duplicates by checking existing tickets for the same week.
  • Auditability: produces logs for created/skipped/failed tickets for admin review.

Architecture breakdown

The following steps describe the execution flow shown in the architecture diagram above:

  1. ScriptRunner Scheduler
    The job runs automatically on a weekly cadence (e.g., every Monday morning).
  2. Groovy Automation Job
    The Groovy script loads configuration (projects, issue type, templates, rules), and starts the generation workflow.
  3. JQL Scope Discovery
    The job runs JQL searches to identify the targets (projects/boards) and checks if a weekly ticket already exists to prevent duplicates.
  4. Issue Creation (Jira APIs)
    For each target, the job creates a standardized Jira issue with predefined fields, summary format, due date, labels, and assignment rules.
  5. Logging & Audit Output
    The job writes a traceable record of created/skipped/failed issues for admin monitoring and easy troubleshooting.
Outcomes
  • Eliminated manual weekly ticket creation and reduced missed recurring tasks.
  • Improved reporting consistency by enforcing standardized fields across projects.
  • Reduced duplicate issues with “already-created” checks per week.
  • Enabled auditing and faster troubleshooting via structured logs.
