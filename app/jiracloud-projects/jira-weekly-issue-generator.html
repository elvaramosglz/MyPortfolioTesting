Problem

Weekly Jira issues were created manually, often late or inconsistently, leading to missing tasks, broken reporting, and unnecessary admin effort.

Main idea

Automate the creation of standardized weekly Jira issues using a ScriptRunner Groovy job, ensuring consistency, traceability, and zero manual overhead.

Automation script

This ScriptRunner job dynamically builds a Jira issue payload, generates a timestamped summary, and creates the issue via Jira REST APIs while logging the execution result. 

import groovy.json.JsonOutput

def projectKey    = "AS"
def issueTypeName = "Task"

def now    = new Date()
def nowStr = now.format("yyyy-MM-dd HH:mm")

def summary = "Issue created with ScriptRunner - ${nowStr}"

def descriptionText = """
This issue was generated automatically by a ScriptRunner job
as part of the Jira Automation Portfolio.
Created by Elva Ramos.
""".stripIndent()

def payload = [
  fields: [
    project   : [ key: projectKey ],
    summary   : summary,
    issuetype : [ name: issueTypeName ],
    description: [
      type: "doc",
      version: 1,
      content: [[
        type: "paragraph",
        content: [[ type: "text", text: descriptionText ]]
      ]]
    ]
  ]
]

def response = post("/rest/api/3/issue")
  .header("Content-Type", "application/json")
  .body(JsonOutput.toJson(payload))
  .asObject(Map)

if (response.status == 201) {
  logger.info("✅ Demo issue created: ${response.body.key}")
} else {
  logger.warn("❌ Demo issue was not created. Status: ${response.status}")
}
