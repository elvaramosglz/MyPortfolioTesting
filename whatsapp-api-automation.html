Production Automation System
Problem

Small businesses often rely on manual WhatsApp conversations to take orders, which leads to inconsistent information, missing details, repetitive questions, and a high operational load. Free-text messages arrive out of order, forcing the business owner to constantly interpret, validate, and confirm orders manually.

Core Idea

Build a WhatsApp bot that automates order intake end-to-end, guiding the user through a structured flow while still allowing natural free-text input, and ensuring that every order reaches the business in a complete, validated, and actionable format.

System Architecture

The following steps describe the production flow shown in the diagram above:

  1. A customer sends a WhatsApp message, which is received by Twilio.
  2. Twilio forwards the message to a FastAPI webhook deployed on Railway.
  3. The application routes the message based on the user’s current conversation state.
  4. Customer data, orders, and state are persisted in a PostgreSQL database.
  5. Once an order is confirmed, an admin notification is sent via WhatsApp.
State Machine

The conversational logic is implemented as a finite state machine. Each user is always in exactly one state, and every incoming message is interpreted according to that state. This prevents ambiguous inputs and guarantees a predictable, step-by-step ordering flow.

Compared to traditional if/else-based logic, the state machine enforces context, simplifies validation, and makes the system easier to extend and maintain as new features are added.
