WhatsApp Bot API

Problem

Many small businesses still take orders through fully manual WhatsApp chats. During peak demand, conversations become fragmented and hard to track: details arrive out of sequence, critical information is missing (items, quantities, delivery time, address), and confirmations vary by operator. This leads to avoidable mistakes, slow response times, and higher operational workload— especially when one person manages both sales and fulfillment.

Common pain points
  • Incomplete orders due to missing required fields.
  • Manual copy/paste into notes or spreadsheets.
  • Inconsistent confirmation messages and follow-ups.
  • Low visibility for the admin when multiple chats happen at once.
Main idea

Build a production-ready WhatsApp conversational bot that automates order intake end-to-end. The bot guides customers through a structured flow, captures free-text inputs, validates required fields, confirms the final order, and sends the admin a clean, standardized summary— reducing errors while improving speed and consistency.

What the bot handles
  • Guided flow: menu → quantities → delivery or pickup → payment → confirmation.
  • Validations: detects missing or invalid values and requests corrections.
  • Clarifications: asks follow-up questions for missing sizes (e.g., 250g/500g, 10/20).
  • Order confirmation: user confirms before saving and notifying admin.
  • Persistence: stores customers, orders, and delivery address for continuity.
  • Admin tools: supports admin commands (confirm, reject, in transit, delivered).
Architecture overview
Architecture breakdown
  • Customer (WhatsApp User): End user interacting with the system through natural, unstructured WhatsApp messages.
  • Twilio WhatsApp Messaging: Messaging gateway that forwards inbound messages to the webhook and delivers bot replies back to the user.
  • Webhook API (FastAPI on Railway): The backend service that receives messages, routes conversation state, validates inputs, and orchestrates the flow.
  • Conversation Router (State Machine): Controls step-by-step conversation logic, including retries, clarifications, and fallbacks.
  • Database (PostgreSQL): Stores customers, orders, and session-related data for traceability and operational continuity.
  • Admin Notifications: Sends structured WhatsApp alerts when orders are confirmed or when key events require admin visibility.
Outcomes
  • Reduced manual back-and-forth by enforcing a structured order flow.
  • Improved order quality by validating required fields and prompting clarifications.
  • Enabled faster operations through instant admin notifications and standardized summaries.
