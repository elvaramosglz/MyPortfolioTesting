Production Automation System
Problem

Small businesses often rely on manual WhatsApp conversations to take orders, which leads to inconsistent information, missing details, repetitive questions, and operational overload. Free-text messages arrive out of order and require constant manual validation.

Core Idea

Build a WhatsApp bot that automates the entire order intake process, guiding users through a structured flow while still allowing natural free-text input, ensuring that every order is complete, validated, and ready for execution.

System Architecture

The following steps describe the production flow shown in the diagram above:

  1. A customer sends a WhatsApp message, received by Twilio.
  2. Twilio forwards the message to a FastAPI webhook.
  3. The application routes the message based on the user’s current state.
  4. Orders and customer data are stored in a PostgreSQL database.
  5. Once confirmed, an admin notification is sent via WhatsApp.
State Machine

The conversational logic is implemented as a finite state machine. Each incoming message is interpreted according to the user’s current state, ensuring context-aware validation, predictable flow, and resilience against out-of-order inputs.
