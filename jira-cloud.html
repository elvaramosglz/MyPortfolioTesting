Jira Cloud Automation

I design and implement automation for Jira Cloud using Groovy (ScriptRunner) and Python (REST API), focusing on reducing manual work, improving data integrity, and supporting large-scale migrations.

Groovy Scripts · ScriptRunner for Jira Cloud

These scripts run directly in Jira Cloud using ScriptRunner. They are ideal for workflow automation, validations, and scheduled jobs inside the Atlassian ecosystem.

  1. Weekly Issue Generator →
    Scheduled job that automatically creates a demo issue in my Automation Showcase (AS) project. It demonstrates scheduled automation, REST API calls, and logging best practices.
    • Creates a new issue once per day (or on demand).
    • Uses Groovy + ScriptRunner + /rest/api/3/issue.
    • Adds a portfolio footer in the issue description for recruiters.

Python Scripts · Jira Cloud REST API

These examples use Python to call the Jira Cloud REST API for administration, reporting, and bulk operations. They complement the Groovy/ScriptRunner work with configuration inventory and data extraction.

  1. Jira Cloud Custom Fields Inventory →
    Python script that connects to Jira Cloud and builds a full inventory of custom fields, their contexts, and their option values, exporting everything to JSON and CSV.
    • Uses /rest/api/3/field, field context, and option endpoints.
    • Resolves option lists via context APIs and editmeta fallback.
    • Runs via GitHub Actions with Jira credentials stored as encrypted secrets.
  2. (Planned) Additional automation utilities:
    • Create and update custom fields via REST.
    • Bulk update issues based on CSV or business rules.
    • Export workflows, permission schemes, and configuration for documentation.
